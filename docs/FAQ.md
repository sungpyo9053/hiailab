# 🤔 자주 묻는 질문 (FAQ)

---

## Q. API 키가 꼭 필요한가요?

**필수는 아닙니다.** 키가 없어도 자판기는 MOCK 결과로 끝까지 동작합니다.
실제 AI 결과를 보고 싶을 때만 OpenAI 키를 발급해서 `/setup` 페이지에 저장하면 됩니다.

---

## Q. 돈이 드나요?

- **HI AI LAB 자체는 무료**입니다. (오픈소스)
- **OpenAI API** 는 사용한 만큼 과금됩니다.
  - 답장 자판기 1회: 보통 ₩1 미만
  - 회의록 자판기 1회: 보통 ₩5 ~ ₩30
  - 카피 자판기 1회: 보통 ₩2 ~ ₩10
- **SMTP 이메일 발송** 은 무료. (본인 Gmail 계정 사용량 한도 내)
- **카카오톡 나에게 보내기** 는 무료.

---

## Q. 내 메일이 개발자(만든 사람) 서버로 전송되나요?

**아니요. 절대 아닙니다.**

HI AI LAB은 **self-hosted** 입니다. 본인이 받은 코드를 본인 컴퓨터/서버에서 직접 실행합니다. 본인이 입력한 내용은:

1. 본인의 서버 (HI AI LAB이 돌고 있는 곳)
2. (실제 모드일 때만) OpenAI API
3. (이메일 보낼 때만) Gmail SMTP 서버
4. (카톡 보낼 때만) 카카오 API

이 4곳 외에는 어디로도 가지 않습니다. HI AI LAB을 만든 사람의 서버는 사이에 끼어들지 않습니다.

---

## Q. Gmail이 읽음 처리되나요?

**현재 버전에서는 Gmail 받은편지함을 읽는 기능 자체가 없습니다.**
SMTP 발송만 지원합니다.

다음 단계에서 Gmail 자동 분석 기능이 추가될 예정인데, 그때도 **읽음 처리는 하지 않도록** 설계됩니다. (Gmail API의 `format=raw` + `metadata.modify` 없이 호출)

---

## Q. 답장이 자동으로 보내지나요?

**아니요.**

- 자판기는 답장 **초안**을 만들 뿐입니다.
- "✉️ 내 이메일로 보내기" 는 결과를 **본인 이메일로** 보내는 것이지, 원래 발신자에게 자동 답장하는 게 아닙니다.
- 다음 단계에서 Gmail 임시보관함(Drafts) 자동 생성 기능이 추가되더라도, **최종 발송은 사람이 직접** 합니다.

---

## Q. 카카오톡으로 다른 사람에게 보낼 수 있나요?

**아니요. 본인의 카톡 "나와의 채팅" 에만 보낼 수 있습니다.**

친구/고객 발송은 카카오의 별도 검수와 비즈 채널이 필요합니다. 현재 버전 범위에는 포함되지 않았습니다.

---

## Q. 네이버 메일도 되나요?

**됩니다.** [SETUP_GMAIL.md](./SETUP_GMAIL.md) 의 "다른 메일 서비스" 표 참고.

```env
SMTP_HOST=smtp.naver.com
SMTP_PORT=587
SMTP_USER=myname@naver.com
SMTP_PASS=네이버메일에서_발급한_비밀번호
```

네이버는 메일함 설정에서 **"POP3/SMTP 사용"** 을 먼저 활성화해야 합니다.

## Q. GitHub 에 올리면 안 되는 파일은 무엇인가요?

**절대 올리면 안 되는 파일:**

| 파일/폴더 | 이유 |
| --- | --- |
| `.env.local` | API 키, 비밀번호 직접 노출 |
| `.env*.local` | 위와 동일 |
| `.hiailab/` | `/setup`에서 저장한 암호화 설정 (암호화돼 있지만 키와 함께 두면 무의미) |
| `node_modules/` | 용량 + 불필요 |

이 모든 것은 이미 `.gitignore` 에 등록되어 있어 일반적인 `git add .` 으로는 올라가지 않습니다. **다만 강제로 추가하면 올라가니** 주의하세요.

> 실수로 키를 커밋했다면, 되돌리는 것만으로는 부족합니다. 반드시 OpenAI/Gmail/카카오에서 해당 키를 **즉시 폐기**하세요.

## Q. dev 서버를 껐다 켜야 하는 경우는?

- `.env.local` 을 수정했을 때
- 코드를 `git pull` 로 업데이트했을 때
- 새 패키지를 설치했을 때 (`npm install`)

`/setup` 페이지에서 저장한 값은 **재시작 없이 즉시 반영**됩니다.

---

## Q. "Module not found" 또는 "Cannot find module" 에러가 나요

`node_modules` 가 망가졌거나 설치가 덜 된 상태입니다.

```bash
rm -rf node_modules package-lock.json
npm install
```

---

## Q. 포트 3000 이 이미 사용 중이라고 나와요

다른 프로그램이 3000 포트를 쓰는 중입니다. 다른 포트로 실행:

```bash
PORT=4000 npm run dev
```

→ <http://localhost:4000>

---

## Q. Docker 빌드가 너무 오래 걸려요

처음 빌드는 의존성 다운로드 때문에 3~5분 걸립니다. 두 번째부터는 캐시 덕분에 빠릅니다.

```bash
docker compose up -d --build
```

## Q. AI 결과가 너무 짧거나 마음에 안 들어요

- 입력을 더 자세하게 적어보세요. ("회의에서 출시일을 4월로 정함" 보다 "참석자 A/B/C, 출시일 4/2 확정, A가 디자인 담당" 이 결과가 좋습니다.)
- 결과를 그대로 쓰지 마시고 **AI는 초안용** 으로 사용하세요. 마지막은 사람이 다듬는 게 정석입니다.
