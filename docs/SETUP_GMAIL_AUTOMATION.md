# 🤖 Gmail 자동 답장 — Google OAuth 등록 가이드

`/agent` 페이지에서 Gmail 받은편지함을 자동으로 읽고 임시보관함에 답장 초안을 만들려면, 본인 명의의 Google OAuth 앱이 한 번 필요합니다.

> ⚠️ 셀프호스팅이라 **각 사용자가 본인 Google Cloud 프로젝트를 만들어** OAuth 클라이언트 ID/시크릿을 발급받습니다.
> 본인 계정에서만 사용하므로 Google 검수 없이 즉시 사용 가능합니다 (External + Testing 모드).

> 💡 이 가이드는 **20분** 정도 걸립니다. 한 번만 설정하면 끝.

---

## 1. Google Cloud Console 들어가기

1. <https://console.cloud.google.com/> 접속 → Google 계정으로 로그인
2. 처음이면 약관 동의 화면이 한 번 뜹니다. 동의.

---

## 2. 새 프로젝트 만들기

1. 좌측 상단 프로젝트 선택 드롭다운 → **새 프로젝트**
2. 프로젝트 이름: `hi-ai-lab` (자유)
3. **만들기** 클릭 → 1분 대기 → 만든 프로젝트 선택

---

## 3. Gmail API 활성화

1. 좌측 메뉴 → **API 및 서비스 → 라이브러리**
2. 검색창에 `Gmail API` 입력 → 결과 클릭
3. **사용** 버튼 클릭 → 1~2분 대기

---

## 4. OAuth 동의 화면 설정

1. 좌측 메뉴 → **API 및 서비스 → OAuth 동의 화면**
2. **User Type**: **외부 (External)** 선택 → **만들기**
3. 다음 항목 입력:
   - 앱 이름: `HI AI LAB`
   - 사용자 지원 이메일: 본인 Gmail
   - 개발자 연락처 이메일: 본인 Gmail
4. **저장 후 계속**
5. **범위 추가 또는 삭제** 클릭
   - 다음 3개 scope에 체크:
     - `https://www.googleapis.com/auth/gmail.readonly`
     - `https://www.googleapis.com/auth/gmail.compose`
     - `https://www.googleapis.com/auth/userinfo.email`
   - **업데이트** → **저장 후 계속**
6. **테스트 사용자**:
   - **+ ADD USERS** → 본인 Gmail 주소 입력 → **저장 후 계속**
   - (이 단계가 중요! 테스트 사용자로 등록된 계정만 사용 가능)
7. 요약 → **대시보드로 돌아가기**

> 검수 없이 쓰려면 **앱을 게시 상태로 두지 마세요.** "테스트" 상태 그대로 두면 본인 등 테스트 사용자만 접근 가능합니다.

---

## 5. OAuth 클라이언트 ID 발급

1. 좌측 메뉴 → **API 및 서비스 → 사용자 인증 정보**
2. 상단 **+ 사용자 인증 정보 만들기 → OAuth 클라이언트 ID**
3. 다음 입력:
   - **애플리케이션 유형**: **웹 애플리케이션**
   - **이름**: `HI AI LAB local`
   - **승인된 리디렉션 URI** → **+ URI 추가**:
     - 로컬: `http://localhost:3000/api/gmail/callback`
     - 서버: `https://본인도메인/api/gmail/callback` (서버 배포 시 함께 등록)
4. **만들기**
5. 팝업에 표시되는 **클라이언트 ID** 와 **클라이언트 보안 비밀** 두 값을 복사

> ⚠️ 보안 비밀은 한 번만 표시됩니다. 안전한 곳에 임시 저장.

---

## 6. HI AI LAB 에 등록

### 방법 A — `/setup` 페이지에서

<http://localhost:3000/setup> 으로 가서 **GOOGLE_OAUTH_CLIENT_ID** 와 **GOOGLE_OAUTH_CLIENT_SECRET** 칸에 붙여넣고 **저장**.

(현재 /setup 화면에는 두 칸이 자동 노출되지 않을 수 있습니다. 그러면 방법 B 사용.)

### 방법 B — `.env.local` 에 직접

```env
GOOGLE_OAUTH_CLIENT_ID=123456-abc.apps.googleusercontent.com
GOOGLE_OAUTH_CLIENT_SECRET=GOCSPX-xxxxx
```

저장 후 dev 서버 재시작 (`Ctrl + C` → `npm run dev`).

---

## 7. `/agent` 페이지에서 Gmail 계정 연결

1. <http://localhost:3000/agent> 접속
2. **"📨 Gmail 계정 연결하기"** 클릭
3. Google 로그인 → 본인 계정 선택
4. "이 앱은 확인되지 않았습니다" 경고가 뜨면 → **고급 → 안전하지 않은 페이지로 이동(이름)** 클릭
   - 본인이 만든 앱이라 안전합니다. (테스트 사용자만 진입 가능)
5. 권한 동의:
   - Gmail 메시지/설정 보기 (받은편지함 읽기)
   - 이메일 메시지 만들기 (임시보관함 초안 생성)
   - 본인 이메일 주소 보기
6. 동의하면 자동으로 `/agent?connected=1` 로 돌아옴

✓ **"Gmail 연결 완료"** 메시지가 보이면 성공.

---

## 8. 자동화 시작

`/agent` 페이지에서:
- **"ON으로 켜기"** 클릭 → 백그라운드에서 5분마다 받은편지함 새 메일 확인
- **"지금 1회 실행"** 클릭 → 즉시 한 번 폴링 (확인용)

처리된 메일은 페이지 하단 **"최근 처리 로그"** 에 카테고리(답장 필요/뉴스레터/광고/알림 등)와 함께 표시됩니다.
**답장 필요** 로 분류된 메일에 대해서만 **Gmail 임시보관함** 에 답장 초안이 자동 생성됩니다.

## 자주 묻는 질문

### Q. "이 앱은 확인되지 않았습니다" 경고가 무서워요
본인이 만든 OAuth 앱이라 안전합니다. Google이 모든 외부 앱에 대해 검수 전까지 띄우는 표준 경고이며, **테스트 사용자에 본인 계정을 등록**해 두었기 때문에 본인만 접근 가능합니다.

### Q. 다른 사람 메일도 처리할 수 있나요?
**아니요.** 본인이 OAuth로 연결한 본인 Gmail만 처리됩니다. 다른 사람의 메일을 보거나 답장하지 않습니다.

### Q. 받은편지함이 자동으로 "읽음" 처리되나요?
**아니요.** `gmail.readonly` scope는 메일을 가져올 뿐 읽음 상태를 변경하지 않습니다. 본인이 직접 메일을 열어보기 전까지 "안 읽은 메일"로 남습니다.

### Q. 답장이 자동으로 발송되나요?
**아니요. 절대 자동 발송되지 않습니다.** `gmail.compose` scope는 임시보관함(Drafts) 생성까지만 가능합니다. 발송은 본인이 Gmail 임시보관함에서 직접 검토 후 보내야 합니다.

### Q. AI가 잘못된 분류를 했어요
"답장 필요" 외 카테고리로 분류된 메일은 그대로 받은편지함에 남고, 임시보관함에는 아무것도 안 만들어집니다. "답장 필요"로 잘못 분류해서 만든 초안은 임시보관함에서 그냥 삭제하시면 됩니다 (어차피 자동 발송 안 됨).

### Q. 토큰이 만료되면?
액세스 토큰은 자동 갱신됩니다 (리프레시 토큰 사용). 다만 Google이 90일 이상 사용 안 한 앱의 리프레시 토큰을 무효화할 수 있습니다. 그때는 `/agent` 에서 **연결 해제 → 다시 연결**.

### Q. 본인 컴퓨터에 저장된 데이터는?
- `.hiailab/config.enc.json` — Client Secret, Refresh Token (AES-256-GCM 암호화)
- `.hiailab/processed.json` — 처리한 메일 ID 목록 + 분류 결과 (메일 본문은 저장 안 함)
- `.hiailab/agent-state.json` — 자동화 ON/OFF, 마지막 실행 시각

세 파일 모두 `.gitignore`에 등록되어 있어 Git에 커밋되지 않습니다.
